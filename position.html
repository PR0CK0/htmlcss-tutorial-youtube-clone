position: static is default position always on back, fixed means it doesn't move on scroll

making something fixed changes it from, for instance, rendering left/right over the whole page, to the bare minimum of the element in it.

for headers, setting a fixed element's left: and top: styles to 0 makes it flush with the browser edge.

position attributes, left and right, top and bottom, if set opposite, i.e., left:50;right:50;, makes the element stretch across the screen; 0 left 0 right means full stretch; set top, left and right, then a height, and you have a nice header setup

style your body with a margin-top or padding-top to make the header sit above site content

same thing with a sidebar, just padding-left on body; set top (to same value as header height if you have one, otherwise zero), left and bottom, the a width and you're done.

absolute position is kinda wacky... it can be scrolled off the page... they are placed on the PAGE, unlike fixed/static, which are on the browser WINDOW

if you set the z-index style rule of the header to 1, then the absolute thing will render below it... default z-index is 0. also if you just put the absolute thing BEFORE the header in the actual HTML file, it parses it first and it gets overriden by the header

absolutes can be put into divs and inherit top, right, left, bottom etc. from that div...

absolutes are useful because lining up a corner is trivial... useful for exit buttons or adding little things into elements

relative position on an element means any sub-element that is ABSOLUTE will be relative to the container... very useful and still allows the item to scroll with the window normally.
